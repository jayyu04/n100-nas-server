# n100-nas-server
# 🖥️ N100 自建家用 NAS 與遠端服務整合系統

本專案以 Intel N100 迷你主機為核心，自行建置一套低功耗、可 24 小時運作的家用 NAS 與遠端服務整合系統，整合個人雲端儲存、資料備份、多裝置同步、遠端存取、自架服務、基礎網路安全，以及未來智慧家庭與本地 AI 延伸應用的系統基礎。

此專案並非單純的儲存設備建置，而是以「個人基礎設施（Personal Infrastructure）」為概念，從真實使用需求出發，實際規劃與整合 Linux Server、Networking、Docker、自架服務、資料備援、供電保護與遠端存取機制，建立一套可長期運作、可持續擴充、可實際解決問題的家用伺服器環境。

---

## 📌 專案概述

本系統使用 Ubuntu Server 24 作為核心作業系統，搭配 mdadm RAID1、APC UPS、Docker / Docker Compose、Nextcloud、Pi-hole、Tailscale、Cloudflare Tunnel 等技術，建構一套兼具資料安全、低功耗、遠端連線、服務擴充與長期可維護性的家用 NAS 與遠端服務平台。

除基本儲存功能外，本系統也實際扮演以下角色：

- 個人雲端與跨裝置同步中心
- 家庭共享備份平台
- 遠端存取與系統管理中樞
- 自架服務部署與測試環境（Homelab）
- 未來智慧家庭與本地 AI 延伸架構基礎

本專案的價值不只在於「架好一台 NAS」，而在於將其作為一套完整的系統基礎設施來設計，從硬體成本、資料安全、網路架構、遠端使用情境到未來擴充性，皆納入整體規劃。

## 🎯 專案動機

在大學專題期間，我已自行組裝一台具備 RTX 3060 與 32GB RAM 的 Windows 桌機，而日常主要使用設備則為 2020 年的 M1 MacBook；此外，租屋處本身也已配置 Apple Home 智慧家庭設備。基於這樣的環境，我開始思考如何整合現有硬體資源，建立一套兼具資料儲存、遠端操作、檔案同步、服務部署與未來智慧家庭延伸應用的個人系統中樞。

另一方面，身為學生，預算是一項必須考量的現實因素。相較於直接購買市售商用 NAS，我選擇自行購買 N100 主機平台與二手 WD Red 硬碟組裝系統，整體成本約新台幣 14,000 元，低於市售成品 NAS 約 20,000 至 30,000 元的價格。這樣的做法不僅較具成本效益，也讓我能更高程度地掌握硬體選型、系統架構與服務配置。

此外，隨著手機資料、專題檔案、生活文件與多裝置同步需求增加，我希望建立一套可自行掌控的個人雲端環境，降低對大型商業雲端訂閱服務的依賴，在資訊安全、經濟成本、資料自主性與長期可維護性之間取得平衡。同時，也希望透過 Ubuntu Server 的實際建置與管理，熟悉 Linux 系統、網路應用與基礎維運思維。

## ✅ 專案目標

本專案的主要目標如下：

- 建立可 24 小時穩定運作的家用 NAS 與遠端服務中樞
- 提供個人雲端平台，支援手機備份、檔案同步與多裝置存取
- 透過 RAID1 提升資料備援能力，降低單碟故障風險
- 透過 UPS 降低突發斷電對資料與系統造成的影響
- 使用 Docker / Docker Compose 建立可擴充、易維護的自架服務環境
- 以 Tailscale / SSH 作為主要遠端管理方式
- 透過 Cloudflare Tunnel + Domain 提供家庭成員可用的外部存取入口
- 建立未來可延伸至智慧家庭整合、本地 AI 模型部署與 GPU Server 串接的系統基礎

##🔐 核心功能
1. RAID1 資料備援
本系統透過 mdadm 建立 RAID1，使用兩顆 WD Red 硬碟進行資料鏡像，降低單一硬碟故障造成資料遺失的風險。對於長期儲存專題資料、生活檔案、手機備份與跨裝置同步內容而言，這是系統穩定性與資料安全性的核心基礎。

2. UPS 不斷電保護
由於 NAS 屬於長時間運作設備，若遇到突發斷電，可能導致資料損毀、檔案系統異常或服務不正常關閉。因此本系統整合 APC UPS，作為基礎供電保護機制，以降低非預期斷電造成的風險，強化 24/7 運作環境下的可靠性。

3. 個人雲端與多裝置同步
系統部署 Nextcloud 作為個人雲端平台，支援：
手機照片與檔案備份
多裝置資料同步
個人文件集中管理
家庭成員共享備份空間
這使 NAS 不只是儲存設備，而是真正承擔日常資料管理與跨裝置同步的核心角色。

4. 遠端存取與個人運算中樞
本系統主要透過 Tailscale + SSH 作為日常遠端存取與伺服器管理方式，使我能在外部網路環境下安全地連回家中設備。此架構也與既有的 Windows 高效能桌機形成整合：透過遠端喚醒與遠端操作方式，我可在外使用 Mac 遠端回 Windows 桌機，提升既有硬體資源利用率，降低重新購買高效能筆電的需求。

5. 外部服務穿透與家庭共享
除個人使用外，我亦申請網域並搭配 Cloudflare Tunnel 建立外部存取入口，讓家人可使用 Nextcloud 進行資料備份。這使整套系統從個人設備進一步延伸為多人可使用的家庭共享雲端平台。

6. Docker 化自架服務管理
本系統使用 Docker / Docker Compose 管理服務，提升部署一致性、維護效率與未來擴充彈性。已部署或測試的服務包含：
Nextcloud
Pi-hole
Home Assistant
Xray / Reality
透過容器化方式，系統更容易進行服務隔離、啟停控制、配置調整與後續擴充。

7. 智慧家庭延伸能力
本專案初期構想之一，是讓 NAS 作為智慧家庭延伸中樞。後續亦部署 Home Assistant，作為與 Apple Home / HomeKit 整合的實驗基礎。雖然目前尚未完全發展為完整的智慧家庭控制中樞，但整體架構已預留未來延伸空間。

##🧩 工程問題與解決方式
問題一：如何在有限預算下建置可長期使用的家用伺服器？
挑戰：商用 NAS 成本較高，對學生而言投資門檻較大；若完全依賴商業雲端服務，則長期成本與資料掌控度也未必理想。
解法：選擇 N100 平台搭配二手 WD Red 硬碟自行組裝，兼顧低功耗、低成本與足夠的服務承載能力。此方式不僅降低建置成本，也讓我能自行掌握硬體規格、儲存策略與系統架構。

問題二：如何兼顧資料安全與長期穩定性？
挑戰：家用伺服器若作為實際備份與同步中心，必須考慮硬碟故障與突然斷電等風險。
解法：透過 mdadm RAID1 建立資料鏡像備援，並整合 APC UPS 作為供電保護機制，從儲存層與供電層同時提升系統穩定性，使系統更接近可持續運作的基礎設施，而不只是能開機使用的主機。

問題三：如何讓遠端連線既穩定又實用？
挑戰：我曾嘗試使用 Xray / Reality 自建 VPN / Proxy 架構，希望在跨境或外部網路環境下回連台灣網路；然而在實際情境中，部分台灣 IP 在中國地區可能受到封鎖，使理論可行與實務穩定之間存在差距。
解法：最終在日常使用上改以 Tailscale 作為主要遠端方案，使遠端 SSH、設備互聯與管理流程更穩定，也降低自建 VPN 的維護複雜度；同時保留 Xray / Reality 作為技術探索與網路實驗成果。

問題四：如何讓 NAS 不只是儲存設備，而是實際可用的家庭系統中樞？
挑戰：許多 NAS 專案最後只停留在「可以存檔」，若缺乏服務整合與實際使用情境，系統價值有限。
解法：我將 NAS 擴充為：個人雲端備份中心多裝置同步平台家庭共享資料空間遠端存取中樞Docker 自架服務實驗環境這使系統真正融入日常學習與生活，而不是只是一台閒置設備。

問題五：如何保留未來擴充性？
挑戰：原先曾規劃將本地 AI 模型與智慧家庭控制功能整合到 NAS / HomePod 架構中，但畢業後花了一年時間準備研究所筆試，因此相關延伸開發暫時停擺。
解法：在系統設計上仍保留未來擴充方向，包括：本地 AI 模型部署更高效能 GPU Server 串接智慧家庭控制中樞監控與儀表板系統也就是說，這套 NAS 架構並非一次性作品，而是一個可持續成長的基礎平台。
